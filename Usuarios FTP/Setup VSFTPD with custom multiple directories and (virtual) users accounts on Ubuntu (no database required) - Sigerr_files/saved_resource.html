                                                                                                          Daniel 3 years ago

                                                                                                          Thanks a bunch. This helped me a lot. I did run into a problem, and I solved it - sort of, at least - so I wanted to bring it to other readers' attentions:

                                                                                                          This guide does give the admin user full access to the system, but it does not give it full permissions. That is, it will be restricted from uploading/deleting/modifying files in places other than the vsftpd home folder and anything contained within it (in my case, /var/www, as I am working with php).

                                                                                                          I tried adding the vsftpd user to the sudoers file, but that did not work (why?). The only workaround I've currently found is changing the permissions on the directory you want to access (and you need to change the permissions individually for every single directory you want to do this with) to so that all users can read, write, and execute from it. So, the command is

                                                                                                          chmod 777 directoryorfilename

                                                                                                          This is, however, insecure in the sense that all virtual users are able to read, write, and execute that directory or file. However, while that may be the case, virtual users cannot access it in the first place, since they are jailed.

                                                                                                          I can't help but think there's a better solution to this, however.

                                                                                                              RDS 3 years ago

                                                                                                              To give 'theadmin' access to files that might be already
                                                                                                              owned do the following: 1. Add this to bottom of vsftpd.conf:
                                                                                                              user_config_dir=/var/www/userconfs (or whatever you want to call
                                                                                                              that directory) 2. Make that directory: mkdir /var/www/userconfs 3.
                                                                                                              Create a specific file in your new directory for 'theadmin': sudo
                                                                                                              nano /var/www/userconfs/theadmin 4. Add this line to the file:
                                                                                                              local_root=/var/www (for access to subdirectories in www) AND this
                                                                                                              line: guest_username=www-data (or whatever the user is that
                                                                                                              currently owns the files you want to edit). 5. Save and sudo
                                                                                                              service vsftpd restart Now you edit and save with your theadmin
                                                                                                              user any files that are created by www-data. Thanks to this
                                                                                                              tuturorial - > http://alethotech.com/?p=309
